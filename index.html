index.html 만들기

Add file → Create new file을 누르고 파일 이름을 소문자로 index.html이라고 적습니다. 아래 코드를 붙여넣으세요.

  <meta charset="utf-8">

  <title>나의 첫 홈페이지</title>
  <style>
    body { max-width:720px; margin:60px auto; padding:24px;
           font-family:sans-serif; line-height:1.7; color:#17324d; }
    h1 { color:#087f73; }
  </style>
</head>
<body>
  <h1>안녕하세요!</h1>
  <p>GitHub Pages로 만든 나의 첫 홈페이지입니다.</p>
</body>
</html>

도메인 구매

내 도메인 고르고 안전하게 구매하기

가비아·카페24·호스팅케이알·Cloudflare Registrar 등에서 원하는 도메인을 검색합니다.

✓ 첫해와 갱신 가격 비교✓ DNS 직접 편집 가능 ✓ 자동 갱신 켜기✓ 계정 2단계 인증
좋은 도메인 고르는 기준
  • 짧고 말로 전달하기 쉬운 이름
  • 하이픈과 숫자를 너무 많이 쓰지 않기
  • 상표권과 비슷한 이름 피하기
  • 구매 전 철자 세 번 확인하기
06
보안

도메인 소유권 확인하기

GitHub 프로필 → Settings → Pages → Add a domain에서 도메인을 추가합니다. GitHub가 보여주는 TXT 레코드를 DNS에 입력하고 Verify를 누릅니다.

왜 하나요?

다른 GitHub 사용자가 내 도메인을 자기 사이트에 연결하는 탈취 위험을 줄입니다. 확인이 끝나도 TXT 레코드는 유지하세요.

전체 흐름을 빠르게 복습하는 요약 이미지입니다. DNS의 루트 A 레코드와 도메인 소유 확인은 위 상세 안내를 기준으로 진행하세요.

막혔을 때 바로 찾기

지금 보이는 증상을 눌러 보세요

초보자가 가장 많이 만나는 문제부터 순서대로 확인합니다.

404페이지를 찾을 수 없다고 나와요
  1. 파일 이름이 소문자 index.html인지 확인
  2. main 브랜치의 맨 위(root)에 있는지 확인
  3. Pages 설정이 main / root인지 확인
  4. Actions의 배포 작업이 완료됐는지 확인
WWWwww 주소만 안 열려요
  1. CNAME 이름이 www인지 확인
  2. 값이 내아이디.github.io인지 확인
  3. https://나 슬래시가 없는지 확인
  4. 기존 www 포워딩 레코드를 정리
ROOTwww 없이 입력하면 안 열려요
  1. 이름 @인 A 레코드가 있는지 확인
  2. GitHub IP 4개를 모두 입력했는지 확인
  3. 예전 A/AAAA 레코드가 충돌하지 않는지 확인
HTTPS자물쇠가 안 생겨요
  1. Pages의 DNS check 완료 여부 확인
  2. 인증서 준비까지 최대 24시간 대기
  3. HTML 안의 http 이미지·스크립트 확인
  4. 필요하면 Custom domain 제거 후 다시 입력
OLD수정 전 화면이 계속 보여요
  1. Commit이 저장됐는지 확인
  2. 배포 완료 후 강력 새로고침
  3. 시크릿 창 또는 다른 기기로 확인
  4. 휴대전화 모바일 데이터로 확인
TAKEN도메인이 이미 사용 중이라고 해요
  1. 다른 내 저장소의 Custom domain 확인
  2. GitHub 계정 Settings → Pages에서 소유 확인
  3. 예전 저장소의 CNAME 연결 제거
  4. 해결되지 않으면 GitHub 공식 문제 해결 문서 확인
⚠
사이트를 삭제하거나 Pages를 끌 예정인가요?

먼저 도메인 판매처에서 GitHub를 가리키는 DNS 레코드를 제거하세요. DNS만 남으면 도메인 탈취 위험이 생길 수 있습니다.

초보자 FAQ

처음이라면 꼭 묻는 질문

QGitHub Pages는 정말 무료인가요?

GitHub Free의 공개 저장소로 정적 홈페이지를 운영할 수 있습니다. 개인 도메인은 별도로 구매하고 갱신해야 합니다.

QREADME 파일을 꼭 만들어야 하나요?

아니요. README는 선택 사항입니다. 홈페이지의 핵심 파일은 게시 폴더 맨 위에 있는 index.html입니다.

Q도메인 없이도 다른 사람에게 보여줄 수 있나요?

네. https://내아이디.github.io 주소만으로 바로 공유할 수 있습니다.

QGitHub Pages로 쇼핑몰이나 회원가입 사이트를 만들 수 있나요?

GitHub Pages는 정적 사이트용입니다. 결제, 비밀번호 저장, 서버 데이터베이스가 필요한 서비스는 별도 백엔드가 필요합니다.

Q도메인을 연결하면 github.io 주소는 사라지나요?

기본 주소는 유지되지만 방문 시 대표 개인 도메인으로 이동할 수 있습니다. Custom domain과 DNS가 올바르게 설정되어야 합니다.

Q도메인 판매처마다 입력 화면이 달라요

메뉴 이름은 달라도 핵심은 “DNS 레코드 관리”입니다. 유형, 이름, 값 세 칸을 이 가이드 표와 맞추세요. 삭제가 필요한 기존 레코드는 판매처 고객센터에 먼저 확인하는 것이 안전합니다.
